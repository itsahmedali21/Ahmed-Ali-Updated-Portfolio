about

Code, Webflow, search, and a good prompt

I'm Ahmed Ali, a frontend developer based in Pakistan. I build interfaces three ways — hand-written code, Webflow, or AI-assisted ("vibe coding") prompts — and make sure whatever ships actually gets found, with solid SEO underneath.

When I write code by hand, it's HTML, CSS and JavaScript, with React when a project calls for it. When I build in Webflow, it's CMS-driven sites clients can update without touching a line of code. When speed matters most, I vibe-code with tools like Claude and Cursor, going from idea to a working build in a fraction of the usual time. And underneath all three, there's SEO — technical audits, Core Web Vitals fixes, and a keyword and internal-linking plan that actually gets the thing found.

Most projects start as a Figma file — or just a prompt — and end as a live site that ranks, not just one that looks good in a portfolio screenshot.
